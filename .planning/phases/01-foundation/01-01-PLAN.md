Named volume pgdata mounted to /var/lib/postgresql/data. Healthcheck: CMD-SHELL with pg_isready -U ${POSTGRES_USER} -d ${POSTGRES_DB}, interval 5s, timeout 5s, retries 10, start_period 10s.

Command: sh -c "alembic upgrade head && uvicorn app.main:app --host 0.0.0.0 --port 8000". Healthcheck: CMD curl -f http://localhost:8000/health, interval 10s, timeout 5s, retries 5, start_period 30s.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host → Docker Compose | .env file read by Docker Compose; secrets available to all services |
| Internet → Docker image build | pip installs packages from PyPI during docker build |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | .env file at repo root | mitigate | .gitignore excludes .env; only .env.example is committed; .env.example contains no real secrets |
| T-01-02 | Tampering | npm/pip/cargo installs | mitigate | slopcheck + blocking human checkpoint (Task 0) for all [ASSUMED] packages before install proceeds |
| T-01-03 | Denial of Service | Docker Compose startup race | mitigate | All services use condition: service_healthy with pg_isready and curl /api/v1/heartbeat; no sleep-based ordering |
| T-01-SC | Tampering | pip package installs | mitigate | Task 0 is a blocking checkpoint requiring human verification of each package source repo before proceeding |
</threat_model>
