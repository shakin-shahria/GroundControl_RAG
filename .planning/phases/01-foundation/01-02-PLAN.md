    Import jwt from PyJWT. Import bcrypt. Import datetime, timezone, timedelta. Import HTTPException, status from fastapi. Import settings from app.core.config.
    Implement hash_password(password: str) -> str:
      password_bytes = password.encode("utf-8")
      if len(password_bytes) > 72: raise ValueError("Password must be 72 bytes or fewer (bcrypt 5.x limit)")
      hashed = bcrypt.hashpw(password_bytes, bcrypt.gensalt())
      return hashed.decode("utf-8")
    Implement verify_password(plain: str, hashed: str) -> bool:
      return bcrypt.checkpw(plain.encode("utf-8"), hashed.encode("utf-8"))
    Implement create_access_token(user_id: int) -> str:
      payload = {"sub": str(user_id), "exp": datetime.now(tz=timezone.utc) + timedelta(minutes=settings.jwt_access_token_expire_minutes)}
      return jwt.encode(payload, settings.jwt_secret_key, algorithm=settings.jwt_algorithm)
    Implement decode_access_token(token: str) -> dict:
      try: return jwt.decode(token, settings.jwt_secret_key, algorithms=[settings.jwt_algorithm])
      except jwt.ExpiredSignatureError: raise HTTPException(status_code=status.HTTP_401_UNAUTHORIZED, detail="Token expired")
      except jwt.InvalidTokenError: raise HTTPException(status_code=status.HTTP_401_UNAUTHORIZED, detail="Invalid token")

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Tampering | JWT decode (algorithm confusion) | mitigate | algorithms=["HS256"] as list in jwt.decode() — never accept alg:none or alg from token header |
| T-01-05 | Information Disclosure | JWT secret key | mitigate | jwt_secret_key read from env var via Pydantic Settings; never hardcoded; no default value forces explicit .env entry |
| T-01-06 | Elevation of Privilege | bcrypt >72 byte truncation | mitigate | len(password_bytes) > 72 check in hash_password() raises ValueError before bcrypt call; enforced in UserCreate schema in Plan 03 |
| T-01-07 | Information Disclosure | Timing attack on password compare | accept | bcrypt.checkpw() is constant-time by design; no additional mitigation needed |
| T-01-08 | Tampering | SQL injection | mitigate | SQLAlchemy ORM parameterizes all queries; raw text() with f-strings is forbidden |
</threat_model>
